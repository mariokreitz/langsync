This walkthrough takes you from a fresh project to a fully working
LangSync setup that validates and syncs your translations.

## Prerequisites

- A TypeScript or JavaScript project with **Node.js 22+**.
- A package manager (**pnpm**, **npm**, or **yarn**).

<Steps>
<Step>

### Install LangSync

Add the CLI as a development dependency.

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm add -D @mariokreitz/langsync
```

</Tab>
<Tab value="npm">

```bash
npm install --save-dev @mariokreitz/langsync
```

</Tab>
<Tab value="yarn">

```bash
yarn add --dev @mariokreitz/langsync
```

</Tab>
</Tabs>

</Step>
<Step>

### Initialize your project

Run the interactive setup. LangSync inspects your `package.json` to
auto-detect your i18n framework, asks a few short questions including the locale file layout, then writes a
typed `langsync.config.ts` and scaffolds empty locale stubs.

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync init
```

</Tab>
<Tab value="npm">

```bash
npx langsync init
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync init
```

</Tab>
</Tabs>

You'll see something like:

```text
  LangSync
  Modern localization workflow tooling.

- Detecting i18n framework...
- Detected: i18next
? Input directory (where locale files live)  > ./src/i18n
? Locale file layout                         > Single file per locale (<input>/<locale>.json)
? Output directory (Excel exports, reports)  > ./translations
? Comma-separated locales                    > en, de, fr
? Default locale                             > en

- Created langsync.config.ts
- Scaffolded src/i18n/en.json
- Scaffolded src/i18n/de.json
- Scaffolded src/i18n/fr.json
```

<Callout type="info">
  Pass `--yes` to skip every prompt and accept the single-file defaults, or `--force` to
  overwrite an existing config.
</Callout>

</Step>
<Step>

### Add your translations

Populate the reference locale (`src/i18n/en.json`) with the source-of-truth
keys for your app:

```json title="src/i18n/en.json"
{
  "greeting": {
    "hello": "Hello",
    "bye": "Goodbye"
  },
  "app": {
    "name": "My App"
  }
}
```

</Step>
<Step>

### Validate

Check every configured locale against the reference. LangSync reports
missing keys, extra keys, and empty values, then exits non-zero on errors —
perfect for CI.

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync validate
```

</Tab>
<Tab value="npm">

```bash
npx langsync validate
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync validate
```

</Tab>
</Tabs>

</Step>
<Step>

### Sync missing keys

Propagate every key from the reference locale into the other locales as
empty placeholders, so translators know what's outstanding:

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync sync
```

</Tab>
<Tab value="npm">

```bash
npx langsync sync
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync sync
```

</Tab>
</Tabs>

Existing translations are preserved. Use `--dry-run` to preview without
writing.

</Step>
<Step>

### Fill the gaps with AI (optional)

Let LangSync translate the empty placeholders for you. Configure a provider in
`langsync.config.ts` (`ai.provider`, defaults to `openai`) and set the matching
API key (for OpenAI, `OPENAI_API_KEY`):

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
OPENAI_API_KEY=sk-... pnpm langsync translate
```

</Tab>
<Tab value="npm">

```bash
OPENAI_API_KEY=sk-... npx langsync translate
```

</Tab>
<Tab value="yarn">

```bash
OPENAI_API_KEY=sk-... yarn langsync translate
```

</Tab>
</Tabs>

Only empty values are filled; your existing translations are never overwritten.
Use `--dry-run` to preview, or `--provider` to override the configured one.

</Step>
<Step>

### Hand off to translators (optional)

Export every locale into a single Excel workbook a non-technical
translator can edit:

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync export excel
```

</Tab>
<Tab value="npm">

```bash
npx langsync export excel
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync export excel
```

</Tab>
</Tabs>

When the workbook comes back:

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync import excel
```

</Tab>
<Tab value="npm">

```bash
npx langsync import excel
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync import excel
```

</Tab>
</Tabs>

</Step>
</Steps>

## What's next

- Wire `langsync validate --reporter json` into your CI — see the
  [validate command reference](./cli/validate) for the exact recipe.
- Automate empty-value translation with [`langsync translate`](./cli/translate).
- Keep locales in sync while you code with [`langsync watch`](./cli/watch).
- Tune your setup, including namespace layouts, with the full [Configuration](./configuration) reference.
- Read the [Integrations](./integrations/i18next) page for your framework.
