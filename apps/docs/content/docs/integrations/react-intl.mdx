LangSync detects [`react-intl`](https://formatjs.io/docs/react-intl/) (and
`@formatjs/intl`) automatically when either is present in your project's
`package.json`.

## Compatibility

| Package          | Version |
| ---------------- | ------- |
| `react-intl`     | >= 6    |
| `@formatjs/intl` | >= 2    |
| React            | >= 18   |

## Try it in 30 seconds

```bash
# 1. LangSync + react-intl
npm install react-intl
npm install -D @mariokreitz/langsync

# 2. Scaffold a typed config (auto-detects react-intl)
npx langsync init

# 3. Check every locale against the reference
npx langsync validate
```

## Locale file layout

`react-intl` resolves message IDs against a **flat** map handed to
`IntlProvider`. LangSync writes nested JSON by default, so when you author
nested files you flatten them to dot-notated IDs at load time (shown in the
wiring example below). If you'd rather skip that step, author flat files
instead. Two strategies are common:

<Tabs items={['Nested (recommended)', 'Flat (legacy FormatJS)']}>
<Tab value="Nested (recommended)">

```json title="src/i18n/en.json"
{
  "checkout": {
    "title": "Checkout",
    "total": "Total: {amount, number, currency/USD}"
  }
}
```

Reference messages with dotted IDs:

```tsx
<FormattedMessage id="checkout.title" defaultMessage="Checkout" />
```

</Tab>
<Tab value="Flat (legacy FormatJS)">

```json title="src/i18n/en.json"
{
  "checkout.title": "Checkout",
  "checkout.total": "Total: {amount, number, currency/USD}"
}
```

Use this when you generate IDs via the FormatJS babel/SWC plugin and want
files to match the plugin's flat output 1:1.

</Tab>
</Tabs>

LangSync handles both — flat keys are written as-is, nested keys are
flattened internally for diffs but written back nested. Pick one and stay
consistent.

## Suggested LangSync config

```ts title="langsync.config.ts"
import { defineConfig } from '@mariokreitz/langsync';

export default defineConfig({
  input: './src/i18n',
  output: './translations',
  locales: ['en', 'de', 'fr'],
  defaultLocale: 'en',
  framework: 'react-intl',
});
```

## Wire react-intl to consume the files

```tsx title="src/main.tsx"
import { IntlProvider } from 'react-intl';
import { useEffect, useState } from 'react';

const SUPPORTED = ['en', 'de', 'fr'] as const;

// react-intl needs a flat id -> message map, so flatten nested JSON on load.
// Skip this step if you already author flat files.
function flatten(obj: Record<string, unknown>, prefix = ''): Record<string, string> {
  return Object.entries(obj).reduce<Record<string, string>>((acc, [key, value]) => {
    const id = prefix ? `${prefix}.${key}` : key;
    if (value && typeof value === 'object') {
      Object.assign(acc, flatten(value as Record<string, unknown>, id));
    } else {
      acc[id] = String(value);
    }
    return acc;
  }, {});
}

async function loadMessages(locale: string) {
  const nested = (await import(`./i18n/${locale}.json`)).default;
  return flatten(nested);
}

export function AppRoot({ children }: { children: React.ReactNode }) {
  const [locale, setLocale] = useState<(typeof SUPPORTED)[number]>('en');
  const [messages, setMessages] = useState<Record<string, string>>({});

  useEffect(() => {
    void loadMessages(locale).then(setMessages);
  }, [locale]);

  return (
    <IntlProvider locale={locale} messages={messages} defaultLocale="en">
      {children}
    </IntlProvider>
  );
}
```

Then reference messages anywhere below the provider, either declaratively with
`FormattedMessage` or imperatively with the `useIntl` hook:

```tsx title="src/components/CheckoutHeader.tsx"
import { FormattedMessage, useIntl } from 'react-intl';

export function CheckoutHeader({ amount }: { amount: number }) {
  const intl = useIntl();
  return (
    <header>
      <h1>
        <FormattedMessage id="checkout.title" defaultMessage="Checkout" />
      </h1>
      <p>{intl.formatMessage({ id: 'checkout.total' }, { amount })}</p>
    </header>
  );
}
```

<Callout title="Result">
  `react-intl` resolves each `id` against the flat message map LangSync keeps in sync. Missing ids
  surface in development as console warnings — and as `missing` issues when you run `langsync
  validate`.
</Callout>

## Recommended workflow

1. Add new `FormattedMessage` usages in your components.
2. Update `src/i18n/en.json` with the new keys (manually or via the
   FormatJS extract step).
3. `langsync sync` propagates the new keys to the other locale files as
   empty placeholders.
4. `langsync validate --reporter json` in CI fails the PR if anything is out
   of sync.
5. `langsync export excel` / `import excel` to hand off bulk translation work.

<Callout title="ICU MessageFormat round-trips losslessly">
  Values such as `{count, plural, one {# item} other {# items}}` are treated
  as opaque strings. LangSync never parses, reformats, or escapes them, so
  every command (sync, export, import) preserves them exactly.
</Callout>
