LangSync detects [`i18next`](https://www.i18next.com) automatically when
any of `i18next`, `react-i18next`, or `i18next-vue` is in your project's
`package.json`.

## Compatibility

| Package         | Version |
| --------------- | ------- |
| `i18next`       | >= 23   |
| `react-i18next` | >= 14   |
| `i18next-vue`   | >= 4    |

## Try it in 30 seconds

```bash
# 1. LangSync + i18next (React example)
npm install i18next react-i18next i18next-http-backend
npm install -D @mariokreitz/langsync

# 2. Scaffold a typed config (auto-detects i18next)
npx langsync init

# 3. Check every locale against the reference
npx langsync validate
```

## Locale file layout

LangSync writes nested JSON, which is i18next's default format:

```text
src/i18n/
  ├ en.json   ← reference (defaultLocale)
  ├ de.json
  └ fr.json
```

```json title="src/i18n/en.json"
{
  "common": {
    "save": "Save",
    "cancel": "Cancel"
  },
  "errors": {
    "required": "{{field}} is required"
  }
}
```

This works directly with the standard
[i18next JSON v4 format](https://www.i18next.com/misc/json-format).

## Wire i18next to consume the files

<Tabs items={['react-i18next', 'i18next (vanilla)', 'i18next-vue']}>
<Tab value="react-i18next">

```ts title="src/i18n/index.ts"
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import HttpBackend from 'i18next-http-backend';

i18n
  .use(HttpBackend)
  .use(initReactI18next)
  .init({
    fallbackLng: 'en',
    supportedLngs: ['en', 'de', 'fr'],
    backend: {
      loadPath: '/locales/{{lng}}.json',
    },
  });

export default i18n;
```

Use it in a component:

```tsx title="src/components/SaveButton.tsx"
import { useTranslation } from 'react-i18next';

export function SaveButton() {
  const { t } = useTranslation();
  return <button>{t('common.save')}</button>;
}
```

Copy the LangSync-managed files into your public folder during build
(e.g. via `vite-plugin-static-copy` or a Next.js `public/` symlink):

```bash
cp src/i18n/*.json public/locales/
```

</Tab>
<Tab value="i18next (vanilla)">

```ts title="src/i18n.ts"
import i18n from 'i18next';
import en from './i18n/en.json' with { type: 'json' };
import de from './i18n/de.json' with { type: 'json' };

await i18n.init({
  fallbackLng: 'en',
  resources: {
    en: { translation: en },
    de: { translation: de },
  },
});

// i18n.t('common.save') === 'Save'
```

</Tab>
<Tab value="i18next-vue">

`i18next-vue` is the maintained Vue 3 binding for i18next (the older
`vue-i18next` package is deprecated — do not confuse it with `vue-i18n`,
which is a separate library with its own file format).

```ts title="src/main.ts"
import { createApp } from 'vue';
import i18next from 'i18next';
import I18NextVue from 'i18next-vue';
import App from './App.vue';
import en from './i18n/en.json';
import de from './i18n/de.json';

await i18next.init({
  lng: 'en',
  fallbackLng: 'en',
  resources: {
    en: { translation: en },
    de: { translation: de },
  },
});

createApp(App).use(I18NextVue, { i18next }).mount('#app');
```

In a template: `<span>{{ $t('common.save') }}</span>`.

## Suggested LangSync config

```ts title="langsync.config.ts"
import { defineConfig } from '@mariokreitz/langsync';

export default defineConfig({
  input: './src/i18n',
  output: './translations',
  locales: ['en', 'de', 'fr'],
  defaultLocale: 'en',
  framework: 'i18next',
});
```

## Recommended workflow

1. Edit `src/i18n/en.json` while developing.
2. Run `langsync sync` to propagate new keys (as empty strings) to other
   locales.
3. Run `langsync validate` in CI to fail PRs that introduce inconsistencies.
4. Hand off `langsync export excel` to non-technical translators.

<Callout title="i18next interpolation is preserved verbatim">
  LangSync treats values as opaque strings, so `{{ field }}` placeholders, ICU MessageFormat, and
  HTML tags all round-trip through `sync`, `export excel`, and `import excel` unchanged.
</Callout>

## Type-safe keys (optional)

i18next can type-check your `t('...')` calls against the reference locale.
Augment the `i18next` module once and your editor will autocomplete keys and
flag typos:

```ts title="src/types/i18next.d.ts"
import 'i18next';
import type en from '../i18n/en.json';

declare module 'i18next' {
  interface CustomTypeOptions {
    defaultNS: 'translation';
    resources: { translation: typeof en };
  }
}
```

Because LangSync keeps `en.json` complete and in sync, these types stay accurate
for every locale.
