LangSync configuration lives at the root of your project. The CLI discovers
it via [cosmiconfig](https://github.com/cosmiconfig/cosmiconfig), so any of
the following are picked up automatically:

- `langsync.config.ts` (recommended — typed)
- `langsync.config.js`
- `langsync.config.mjs`
- `langsync.config.json`
- `"langsync"` field in `package.json`
- `.langsyncrc` / `.langsyncrc.json`

## Recommended: `langsync.config.ts`

The TypeScript file gives you autocomplete and validation in your editor
via the `defineConfig` helper.

```ts title="langsync.config.ts"
import { defineConfig } from '@mariokreitz/langsync';

export default defineConfig({
  input: './src/i18n',
  output: './translations',
  locales: ['en', 'de', 'fr'],
  defaultLocale: 'en',
  framework: 'i18next',
  excel: {
    file: 'translations.xlsx',
    sheetName: 'Translations',
  },
  ai: {
    provider: 'openai',
    model: 'gpt-4o-mini',
  },
});
```

## Options reference

### Top-level

| Option          | Type       | Required | Default                  | Description                                              |
| --------------- | ---------- | -------- | ------------------------ | -------------------------------------------------------- |
| `input`         | `string`   | yes      | —                        | Directory containing per-locale JSON files.              |
| `output`        | `string`   | yes      | —                        | Output directory for Excel exports and reports.          |
| `locales`       | `string[]` | yes      | —                        | List of locale codes you support.                        |
| `defaultLocale` | `string`   | no       | First entry in `locales` | Reference locale used by `validate` and `sync`.          |
| `framework`     | `string`   | no       | `none`                   | One of `i18next`, `ngx-translate`, `react-intl`, `none`. |
| `excel`         | `object`   | no       | see below                | Excel I/O settings.                                      |
| `ai`            | `object`   | no       | see below                | AI translation settings (used by `translate`).           |

### `excel`

| Option      | Type     | Default             | Description                                      |
| ----------- | -------- | ------------------- | ------------------------------------------------ |
| `file`      | `string` | `translations.xlsx` | Path (relative to project root) of the workbook. |
| `sheetName` | `string` | `Translations`      | Worksheet name inside the workbook.              |

### `ai`

| Option     | Type     | Default          | Description                                               |
| ---------- | -------- | ---------------- | --------------------------------------------------------- |
| `provider` | `string` | `openai`         | One of `openai`, `deepl`, `anthropic`, `gemini`.          |
| `apiKey`   | `string` | —                | API key. Falls back to the provider env var when omitted. |
| `model`    | `string` | provider default | Provider model id (e.g. `gpt-4o-mini`).                   |

<Callout title="Provider availability">
  Only `openai` is released today. `deepl`, `anthropic`, and `gemini` are gated behind
  `LANGSYNC_AI_EXPERIMENTAL=1` until they ship.
</Callout>

## Locale file layout

LangSync expects one JSON file per locale at `<input>/<locale>.json`:

```text
src/i18n/
  ├ en.json   ← reference (defaultLocale)
  ├ de.json
  └ fr.json
```

Each file is a free-form nested JSON tree. Keys are dot-notated internally
(`greeting.hello`) so deeply nested structures are supported.

```json title="src/i18n/en.json"
{
  "greeting": {
    "hello": "Hello",
    "bye": "Goodbye"
  },
  "app": { "name": "My App" }
}
```

<Callout title="Missing files are tolerated">
  If a configured locale file doesn't exist on disk yet, LangSync treats it as an empty tree and
  reports every reference key as `missing` (instead of crashing). Run `langsync sync` to scaffold it
  with empty placeholders.
</Callout>

## Resolution order

When multiple config files exist, cosmiconfig's discovery order applies
(top wins):

1. `package.json#langsync`
2. `.langsyncrc` / `.langsyncrc.json`
3. `langsync.config.ts`
4. `langsync.config.mjs`
5. `langsync.config.js`
6. `langsync.config.json`

Stick to one. The first match wins; the rest are ignored.

## Validation

The config is validated with [zod](https://zod.dev) at load time. Invalid
configs fail fast with a clear error pointing to the field at fault:

```text
Error: Invalid LangSync configuration
  - locales: must contain at least one locale
  - defaultLocale: must be one of the configured locales
```
