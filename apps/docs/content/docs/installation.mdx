## Requirements

- **Node.js 22 or newer.** LangSync ships as ESM and targets modern Node.
  Check with `node --version`.
- **A TypeScript- or JavaScript-based project.** LangSync writes a typed
  `langsync.config.ts` by default but `.js`, `.mjs`, and `.json` are
  also supported.

## Install as a dev dependency

LangSync is published on npm as
[`@mariokreitz/langsync`](https://www.npmjs.com/package/@mariokreitz/langsync).
The installed binary is exposed as `langsync` on your `node_modules/.bin`.

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm add -D @mariokreitz/langsync
```

</Tab>
<Tab value="npm">

```bash
npm install --save-dev @mariokreitz/langsync
```

</Tab>
<Tab value="yarn">

```bash
yarn add --dev @mariokreitz/langsync
```

</Tab>
</Tabs>

## One-off usage with `npx`

If you just want to scaffold or run a single command without adding LangSync
to your dependencies:

```bash
npx @mariokreitz/langsync init
```

The `npx` invocation downloads, caches, and runs the latest version.

## After installation

The binary `langsync` is now available in your project. Verify with:

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync --version
```

</Tab>
<Tab value="npm">

```bash
npx langsync --version
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync --version
```

</Tab>
</Tabs>

<Callout title="Why is the package name scoped?">
  The unscoped `langsync` name on npm was taken by an unrelated package. The
  scoped name `@mariokreitz/langsync` lets us publish under our brand. The
  installed binary is still `langsync`, so all your `npx langsync …` commands
  look the same.
</Callout>
