---
title: langsync sync
description: Synchronize translation keys across all configured locales.
---

`langsync sync` copies keys from the reference locale into every other
locale, preserving existing translations and inserting empty placeholders
for keys that do not yet have a translation. Keys that no longer exist in
the reference locale are removed from the targets.

## Usage

```bash
langsync sync [--dry-run]
```

## Flags

| Flag        | Default | Description                                        |
| ----------- | ------- | -------------------------------------------------- |
| `--dry-run` | `false` | Report planned writes without modifying any files. |

## Behavior

- Keys present in the reference but missing in a target are added as `""`.
- Existing target values are preserved; the reference never overwrites them.
- Keys present in the target but missing in the reference are removed.
- The reference locale file itself is never modified.

## Complete example

Before — `de.json` is missing a key the reference has:

```json title="src/i18n/en.json"
{ "greeting": { "hello": "Hello", "bye": "Goodbye" } }
```

```json title="src/i18n/de.json"
{ "greeting": { "hello": "Hallo" } }
```

```bash
$ npx langsync sync
langsync ✔ Updated src/i18n/de.json
```

After — the missing key is added as an empty placeholder, the existing
translation is untouched:

```json title="src/i18n/de.json"
{ "greeting": { "hello": "Hallo", "bye": "" } }
```

Run `langsync translate` next to fill `""` automatically, or hand the file to a
translator via `langsync export excel`.
