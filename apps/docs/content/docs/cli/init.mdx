`langsync init` scaffolds a typed `langsync.config.ts` (or `.json` with
`--json`) and creates locale JSON stubs in your selected layout.

## Usage

<Tabs items={['pnpm', 'npm', 'yarn']}>
<Tab value="pnpm">

```bash
pnpm langsync init [--force] [--json] [--yes]
```

</Tab>
<Tab value="npm">

```bash
npx langsync init [--force] [--json] [--yes]
```

</Tab>
<Tab value="yarn">

```bash
yarn langsync init [--force] [--json] [--yes]
```

</Tab>
</Tabs>

## What it does

1. Inspects `package.json` for known i18n framework dependencies
   (`i18next`, `react-i18next`, `i18next-vue`, `@ngx-translate/core`,
   `@ngx-translate/http-loader`, `react-intl`, `@formatjs/intl`) and
   pre-selects the detected framework.
2. Asks for **input directory**, **locale file layout**, **output directory**,
   **locales**, and **default locale**.
3. If you choose a namespaced layout, asks for initial namespace names.
4. Writes a typed `langsync.config.ts` (or `langsync.config.json` with
   `--json`) at the project root.
5. Scaffolds empty locale files for every configured locale and initial
   namespace. Existing files are preserved.

## Interactive prompts

```text
? Where are your source i18n files? > ./src/i18n
? How should locale files be organized?
  > Single file per locale (<input>/<locale>.json)
    Namespace folders per locale (<input>/<locale>/<namespace>.json)
    Flat namespace-prefixed files (<input>/<locale>.<namespace>.json)
? Where should translation output go? > ./translations
? Which locales? (comma-separated) > en,de
? Default (reference) locale? > en
? Which i18n framework do you use? > i18next
```

For either namespaced layout, `init` asks one additional question:

```text
? Initial namespaces? (comma-separated) > common
```

The namespace list is trimmed and empty entries are ignored. If the answer is
empty, `init` scaffolds `common`.

## Flags

| Flag      | Default | Description                                                            |
| --------- | ------- | ---------------------------------------------------------------------- |
| `--force` | `false` | Overwrite an existing `langsync.config.*` file without asking.         |
| `--json`  | `false` | Emit `langsync.config.json` instead of `.ts`.                          |
| `--yes`   | `false` | Skip every prompt; use the single-file defaults (`./src/i18n`, `en,de`). |

<Callout title="Non-interactive mode stays single-file">
  `langsync init --yes` does not enable namespaces. It preserves the original default layout and
  creates `<input>/<locale>.json` files only.
</Callout>

## Generated layouts

<Tabs items={['Single file', 'locale-dir', 'locale-prefix']}>
<Tab value="Single file">

Default selection. No `namespaces` block is written.

```ts title="langsync.config.ts"
import { defineConfig } from '@mariokreitz/langsync';

export default defineConfig({
  input: './src/i18n',
  output: './translations',
  locales: ['en', 'de'],
  defaultLocale: 'en',
  framework: 'i18next',
});
```

```text
src/i18n/
  en.json
  de.json
```

</Tab>
<Tab value="locale-dir">

Choose `Namespace folders per locale` and enter `common,auth/login`.

```ts title="langsync.config.ts"
import { defineConfig } from '@mariokreitz/langsync';

export default defineConfig({
  input: './src/i18n',
  output: './translations',
  locales: ['en', 'de'],
  defaultLocale: 'en',
  framework: 'i18next',
  namespaces: { structure: 'locale-dir' },
});
```

```text
src/i18n/
  en/
    common.json
    auth/login.json
  de/
    common.json
    auth/login.json
```

## Examples

### First-time setup (interactive)

```bash
pnpm langsync init
```

### Non-interactive (CI / scripted)

```bash
pnpm langsync init --yes
```

Produces a single-file config with:

- `input: './src/i18n'`
- `output: './translations'`
- `locales: ['en', 'de']`
- `defaultLocale: 'en'`
- Framework auto-detected from `package.json` (or `none`)
- No `namespaces` block

### Re-initialize an existing project

```bash
pnpm langsync init --force
```

### Emit JSON instead of TypeScript

```bash
pnpm langsync init --json
```

<Callout title="Existing locale files are never touched">
  `init` only creates files that don't already exist on disk. Your existing translations are safe.
</Callout>

See [Configuration](../configuration) for every supported option.
