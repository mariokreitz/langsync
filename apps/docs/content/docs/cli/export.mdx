## Subcommands

- `langsync export excel` - export to an `.xlsx` workbook.

## `langsync export excel`

Exports every configured locale into a single Excel workbook. Useful for
sharing translation work with non-technical translators.

With `namespaces` configured, export includes every discovered namespace file.
If no namespace files are discovered, the command fails with an actionable
error instead of writing an empty workbook.

### Usage

```bash
langsync export excel [--file <path>] [--sheet <name>]
```

### Flags

| Flag             | Default     | Description                             |
| ---------------- | ----------- | --------------------------------------- |
| `--file <path>`  | from config | Output `.xlsx` path (overrides config). |
| `--sheet <name>` | from config | Worksheet name (overrides config).      |

If `excel.file` / `excel.sheetName` are unset in `langsync.config.ts`, the
defaults are `translations.xlsx` and `Translations`.

### Workbook layout

Single-file projects keep the legacy workbook format. Row 1 is a header row:

| key            | en    | de       |
| -------------- | ----- | -------- |
| greeting.hello | Hello | Hallo    |
| greeting.bye   | Bye   | Tschuess |

- Column A contains the flattened (dot-notated) key.
- Subsequent columns contain values per locale, in the order configured.
- Keys are sorted alphabetically.

Namespaced projects add a leading `namespace` column:

| namespace  | key            | en     | de      |
| ---------- | -------------- | ------ | ------- |
| common     | app.title      | App    | App     |
| auth/login | form.submit    | Sign in | Anmelden |

- Column A contains the namespace.
- Column B contains the flattened key within that namespace.
- Locale columns start after `key`.
- Row identity is `(namespace, key)`; duplicate identities are rejected on
  import.

### Compatibility

- Existing single-file workbooks use `key, <locale...>` and remain supported.
- Namespaced workbooks use `namespace, key, <locale...>`.
- Importing a namespaced workbook into a single-file config errors clearly.
- Importing a legacy workbook into a namespaced config errors clearly.

### Complete example

```bash
$ npx langsync export excel
langsync ✔ Exported 3 locale(s) to translations.xlsx
```

Open `translations.xlsx`, hand it to a translator, and when it comes back run
[`langsync import excel`](./import) to merge the values straight back into your
locale JSON files.
