---
title: langsync validate
description: Validate locale consistency, structure, and missing keys.
---

`langsync validate` compares every configured locale against the reference
locale (`defaultLocale`, or the first entry of `locales` when unset) and
reports inconsistencies.

With `namespaces` configured, validation runs independently per namespace file.

## Usage

```bash
langsync validate [--reporter <pretty|json>]
```

## Flags

| Flag                | Default  | Description                                    |
| ------------------- | -------- | ---------------------------------------------- |
| `--reporter <kind>` | `pretty` | Output format. Use `json` for CI integrations. |

## Issue types

| Type      | Meaning                                                              | Exit code impact |
| --------- | -------------------------------------------------------------------- | ---------------- |
| `missing` | Key is present in the reference locale but absent in another locale. | Fails (exit 1)   |
| `extra`   | Key is present in a locale but not in the reference locale.          | Fails (exit 1)   |
| `empty`   | Key exists but its value is an empty string.                         | Warning (exit 0) |

## Namespaced behavior

- Missing whole namespace files in target locales are treated as empty trees.
- Extra target-only namespace files validate as `extra` while they contain keys.
- Empty extra namespace files left behind by `sync` are inert and do not keep
  validation failing.
- If no namespace files are discovered while `namespaces` is configured, the
  command fails with an actionable error.
- Pretty output labels namespaced keys as `namespace:key`, for example
  `auth/login:greeting.submit`.

## Exit codes

- `0` - no `missing` or `extra` issues (empty values are warnings only).
- `1` - at least one `missing` or `extra` issue, or a runtime error
  (missing config, unreadable file, etc.).

## JSON reporter

```bash
langsync validate --reporter json
```

```json
{
  "referenceLocale": "en",
  "issues": [
    {
      "type": "missing",
      "locale": "de",
      "key": "greeting.bye",
      "namespace": null,
      "path": "src/i18n/de.json"
    }
  ]
}
```

Every issue record includes `namespace` and the resolved file `path`. In
single-file mode `namespace` is `null`; in namespaced mode it holds the
namespace name, so consumers can distinguish the same key in different files.

## Complete example

Given these files:

```json title="src/i18n/en.json"
{ "greeting": { "hello": "Hello", "bye": "Goodbye" } }
```

```json title="src/i18n/de.json"
{ "greeting": { "hello": "Hallo" } }
```

Running validate:

```bash
$ npx langsync validate
langsync ℹ missing  de  greeting.bye
langsync ℹ Summary: 1 missing, 0 extra, 0 empty
```

The command exits with code `1` because `de` is missing `greeting.bye`, so the
PR check fails until the key is added (for example via `langsync sync`).
