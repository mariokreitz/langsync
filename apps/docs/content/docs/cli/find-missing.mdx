---
title: langsync find-missing
description: Find missing translation keys per locale.
---

`langsync find-missing` reports keys that exist in the reference locale but
are missing from one or more other locales. It is a focused subset of
[`langsync validate`](./validate).

With `namespaces` configured, the command checks each discovered namespace
independently.

## Usage

```bash
langsync find-missing [--reporter <pretty|json>]
```

## Flags

| Flag                | Default  | Description                                    |
| ------------------- | -------- | ---------------------------------------------- |
| `--reporter <kind>` | `pretty` | Output format. Use `json` for CI integrations. |

## Namespaced behavior

- Missing whole namespace files in target locales are synthesized as empty
  trees, so every reference key in that namespace is reported as missing.
- Extra target-only namespace files are outside this command's focused report;
  use `langsync validate` to report them as `extra`.
- If no namespace files are discovered while `namespaces` is configured, the
  command fails with an actionable error.
- Pretty output labels namespaced keys as `namespace:key`, for example
  `common:app.title`.

## Exit codes

- `0` - no missing keys.
- `1` - at least one locale is missing one or more keys, or a runtime error.

## JSON reporter

```bash
langsync find-missing --reporter json
```

```json
{
  "referenceLocale": "en",
  "missingByLocale": {
    "de": [{ "namespace": null, "key": "greeting.bye", "path": "src/i18n/de.json" }],
    "fr": [
      { "namespace": null, "key": "app.tagline", "path": "src/i18n/fr.json" },
      { "namespace": null, "key": "greeting.bye", "path": "src/i18n/fr.json" }
    ]
  }
}
```

Each entry carries `namespace`, `key`, and the resolved file `path`. In
single-file mode `namespace` is `null`; in namespaced mode it holds the
namespace name, so automated checks can separate identical keys from
different namespace files.
